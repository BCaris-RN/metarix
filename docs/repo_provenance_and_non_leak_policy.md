# Repo Provenance And Non-Leak Policy

MetaRix may be inspired by prior private work, but all public repo code, docs, assets, schemas, fixtures, and examples must be independently authored for MetaRix.

Public repo material must be expressed in MetaRix-native language. If a concept is reused, rename and re-express it for MetaRix instead of carrying forward prior project names, internal path names, copied manifests, copied screenshots, copied fixtures, or copied semantic artifacts.

Never copy proprietary source files, schemas, manifests, semantic bundles, screenshots, fixtures, or internal path names into this public repo.

Do not introduce prior private repo names, copied path signatures, or legacy markers that are not part of the current MetaRix public repo contract.

Examples of strings that should not appear in newly added public repo content unless they are being discussed inside provenance policy or review notes:

- `Suprema_C`
- `cp_super_stack`
- `legacy_phoenix_ref`
- `governance_job_registry`
- `policy_bundle_manifest.json`

The following materials are prohibited in the public MetaRix repo:

- copied source files or docs from prior private repos
- copied schema or policy bundle artifacts
- copied semantic bundle dumps
- copied screenshots or fixtures from private systems
- copied internal folder names or relative paths from prior projects

MetaRix should remain self-hostable, public-safe, and independently authored. Local runtime data, local secrets, and local-only machine state must stay outside the public repo boundary.
